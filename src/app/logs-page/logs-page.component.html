Setting up an API

2022-02-07

The next thing I'd like to work on is setting up an api. I'll eventually need this for the following projects:
  • Hosting a media server
  • My rubix cube timer project
  • Storing these log posts in a database
Here is the game plan going into the api:
  1. Create Spring boot api locally (I have plenty of experience using the spring initializr so this should be straight forward)
  2. Figure out how to deploy api to AWS According to the internet, I should deploy my api using Elastic Beanstalk)
  3. Configure domain so api can be hosted on api.austinhuynh.com or austinhuynh.com/api (I'm not looking forward to this since I've never been able to do it in the past)
  4. Connect front end to backend
  5. Setup security (perhaps OAuth?)
As expected, setting up the api locally and getting the api deployed to AWS was straight forward. Getting the domain configured to handle a front end and a backend also went as expected: not great. It took me a while to figure out what I wanted to google, which was "how to host multiple origins from cloud front". I learned that there would be more problems for me if I were to use a subdomain like "api.austinhuynh.com" regarding SSL certificates and CORS policies (the worst). Although my preference is "what ever would be easier", I like the "austinhuynh.com/api/*" approach since that's what my work does for a lot of their stuff.
All of that work just so I can see this beautiful response: {{response}}
